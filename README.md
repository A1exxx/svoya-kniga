# СвояКнига — онлайн-бухгалтерия для ИП на УСН (РФ)

Своё веб-приложение бухгалтерии вместо Контур.Эльбы: расчёт налогов и взносов,
первичка, отчётность, зарплата сотрудников, налоговый календарь — для **ИП на
УСН** в России. Цель: сначала для себя (обкатка на реальном ИП с бухгалтером),
потом — продукт.

- **Живое демо:** https://a1exxx.github.io/svoyakniga/ (данные локально в браузере, без входа)
- **Боевой режим на своём компьютере:** двойной клик `start-svoyakniga.bat` (вход по логину/паролю, база — файл на диске). См. [docs/LOCAL-RUN.md](docs/LOCAL-RUN.md)
- **Репозиторий:** A1exxx/svoyakniga (`master` — исходники, `gh-pages` — сборка демо)
- **Статус:** ядро готово и обкатывается; серверный слой (аккаунты+БД) построен, переезд на платный РФ-сервер — по утверждению функционала.

> 📘 **Полный справочник «всё о проекте» → [docs/HANDBOOK.md](docs/HANDBOOK.md)** —
> все экраны, фичи, база данных, движок расчётов, сервер, карта кода.

## Быстрый старт

| Хочу… | Что делать |
|---|---|
| Просто посмотреть | Открыть https://a1exxx.github.io/svoyakniga/ |
| Работать «по-настоящему» на этом ПК | Двойной клик `start-svoyakniga.bat` → браузер на `http://localhost:8011`, создать кабинет |
| Поднять фронтенд для разработки | `cd web && npm install && npm run dev` (порт 5180) |
| Поднять бэкенд (API+расчёты) | `cd backend && .venv\Scripts\python -m uvicorn app.main:app --reload --port 8000` → http://localhost:8000/docs |
| Прогнать все тесты | `cd web && npx vitest run` (фронт) + `cd backend && .venv\Scripts\python -m pytest -q` (бэк) |
| Собрать демо и задеплоить | `cd web && npm run build && npx gh-pages -d dist -b gh-pages --dotfiles` |

Требования: Node.js 18+ и Python 3.12 (для бэкенда). Зависимости бэкенда —
`backend/requirements.txt`.

## Три режима работы

1. **Демо (GitHub Pages)** — статический сайт, данные только в браузере (localStorage
   + зеркало IndexedDB). Без входа, без сервера. Так работает живое демо.
2. **Локальный боевой** — бэкенд FastAPI отдаёт приложение и API с одного адреса,
   вход по логину/паролю, база — SQLite-файл `backend/svoyakniga.db` на вашем
   диске, с историей версий и бэкапами. Запуск — `start-svoyakniga.bat`.
3. **Сервер (облако)** — тот же код на хостинге. Бесплатный тест на фейк-данных
   (Render+Neon, [docs/DEPLOY-FREE-TEST.md](docs/DEPLOY-FREE-TEST.md)); боевой с
   реальными данными — только РФ-хостинг по 152-ФЗ ([docs/SERVER-SETUP.md](docs/SERVER-SETUP.md)).

Серверный режим включается флагом сборки `VITE_SERVER_MODE=true` (или адресом
`VITE_API_BASE`). Без флага — приложение в локальном режиме (демо не меняется).

## Технологии

- **Фронтенд:** Vite + React 19 + TypeScript + Tailwind v3, HashRouter, `base: './'`.
  Расчёты — на клиенте (decimal.js). Иконки — inline-SVG.
- **Бэкенд:** Python + FastAPI. Движок расчётов `taxcore` (зеркало TS-движка,
  паритет проверяется тестами). Аккаунты/БД — SQLAlchemy (SQLite локально, PostgreSQL в проде).
- **Хранение:** локально — localStorage + IndexedDB; на сервере — БД с
  версионированием рабочего стола и многослойными бэкапами.
- **Тесты:** 351 vitest (фронт) + 97 pytest (бэк), property-тесты (fast-check), e2e (Playwright).

## Важное про закон и честность

- **Реальные данные клиентов (ИНН/СНИЛС/паспорта/зарплаты)** — персональные данные
  граждан РФ; по 152-ФЗ их база обязана быть на территории РФ. Бесплатный/зарубежный
  хостинг — только для теста на выдуманных данных. Подробно — [docs/SERVER-SETUP.md](docs/SERVER-SETUP.md).
- **Отправка отчётности в ФНС/СФР, сальдо ЕНС, сверка** — пока честная имитация с
  квитанцией; реально это идёт по ТКС через оператора ЭДО с КЭП (серверный этап,
  [docs/FNS-INTEGRATION.md](docs/FNS-INTEGRATION.md)). Печатные формы и XML —
  готовятся, перед сдачей сверяются с XSD ФНС.
- Все расчёты дублируются в **двух движках** (TS и Python) и сверяются тестами;
  корректность подтверждается многоагентными аудитами против НК РФ.
