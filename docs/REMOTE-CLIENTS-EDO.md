# Удалённое ведение клиента + сдача отчётности с ЭЦП

> Методичка для бухгалтерской конторы: как полностью удалённо вести бухгалтерию и сдавать
> ВСЮ отчётность за клиента из другого региона (кейс: контора в Воронеже, клиент во Владивостоке,
> приехать не может). Актуально на середину 2026. Со ссылками и датами.
>
> Связано с продуктом СвояКнига: сейчас подпись КЭП и отправка — имитация (статусы + квитанция),
> реальная отправка в роадмапе. Этот документ описывает, как сделать «по-настоящему» и под удалёнку.

## 3 факта, которые снимают проблему «клиент не может приехать»

1. Клиенту НЕ нужно ехать к вам. Один раз — в банк В СВОЁМ городе (Сбер/ВТБ и др. — доверенные лица УЦ ФНС) или подпись на смартфон. К вам в Воронеж — никогда.
2. Передавать вам свой токен/пароль клиент НЕ должен — нарушение 63-ФЗ ст. 10 (ответственность на владельце ключа). Легальная замена — МЧД.
3. Минимум действий клиента: 1 раз получить свою КЭП → 1 раз подписать МЧД на вас → дальше всё подписываете и сдаёте вы. Отчётность он больше не трогает.

## ЧАСТЬ 1. Кто подписывает отчёт — 2 модели

| | Модель Б — подпись бухгалтера + МЧД (стандарт рынка) | Модель А — подпись клиента |
|---|---|---|
| Кто подписывает | Вы (КЭП физлица бухгалтера) | Клиент (его КЭП) |
| Что приложено | МЧД от клиента | ничего |
| Участие клиента | нулевое (1 раз выдал МЧД) | каждый отчёт ждёт его подписи |
| Доля рынка | 85–90% аутсорсеров | для тех, кто хочет лично контролировать |
| Минус | вы отвечаете за подписанное; следить за сроком МЧД | клиент = узкое горло, риск пропуска срока |

Для далёкого клиента → Модель Б.

### Модель Б по шагам
1. Бухгалтер получает личную КЭП физлица в коммерческом УЦ (Контур/Астрал/Такском), 1 500–5 000 ₽/год.
   УЦ ФНС выдаёт только директорам/ИП — на сотрудника-представителя только коммерческий УЦ.
2. Клиент создаёт МЧД: сайт ФНС m4d.nalog.gov.ru/emchd (бесплатно, формат 003) или в кабинете оператора.
   Указывает ИНН+СНИЛС бухгалтера, полномочия (код сдачи отчётности или «99»), срок.
3. Клиент подписывает МЧД своей КЭП ФНС → регистрация в реестре ФНС (блокчейн) ~сутки.
4. При сдаче: бухгалтер подписывает своей КЭП → оператор сам прикрепляет МЧД → ФНС принимает.

ВЕДОМСТВА: одной МЧД мало.
- ФНС + Росстат — формат 003.
- СФР с 01.07.2025 — собственный формат МЧД СФР (коды FSS/SFR), формат 003 НЕ принимает.
- → С клиента нужно ДВЕ МЧД (ФНС/Росстат + СФР). Контур.Экстерн и СБИС делают обе за один заход.

### Передоверие (если контора — юрлицо с несколькими бухгалтерами)
Клиент выдаёт МЧД с правом передоверия на ООО-контору → контора раздаёт МЧД сотрудникам.
Цепочка юрлицо→юрлицо→физлицо нотариуса НЕ требует (нотариус только при физлицо→физлицо).
Один бухгалтер → проще выдавать МЧД сразу на него.

## ЧАСТЬ 2. Как клиент получает СВОЮ КЭП, не выезжая

Клиенту нужна одна своя КЭП ФНС (ИП/руководителя), чтобы подписать МЧД. 3 канала:

1. Банк — доверенное лицо УЦ ФНС (универсально). 7 банков: Сбер, ВТБ, Т-Банк, Альфа, ПСБ, ЕЭТП,
   Аналитический центр (аккредитация до конца 2026–2027). Клиент идёт в Сбер/ВТБ во Владивостоке
   (паспорт, СНИЛС, ИНН, чистый токен) → КЭП за 20–30 мин. Токен Рутокен ЭЦП 3.0 (~1 500–2 500 ₽)
   покупает сам (OZON/DNS/банк). Список: https://www.nalog.gov.ru/rn77/related_activities/ucfns/dlucfns/
2. Приложение ФНС «Моя подпись» + биометрия ЕБС (без токена). С августа 2025: УКЭП на смартфон,
   если сдана биометрия в банке-партнёре ЕБС. Минус — работает в основном в экосистеме ФНС; для всех
   операторов надёжнее токенный вариант. Для выдачи МЧД годится. https://www.nalog.gov.ru/ds/
3. Перевыпуск без визита — если КЭП ещё действует, продление в ЛК налогоплательщика. Просрочена → заново.

Госключ: клиент-ИП/директор им МЧД от бизнеса НЕ выпустит (Госключ = УКЭП физлица, а МЧД идёт от ИП/ООО).
Госключ полезен бухгалтеру как личная УКЭП; отчётность напрямую им не сдать.

## ЧАСТЬ 3. Что нужно бухгалтеру + через что сдавать

- Своя КЭП физлица (Контур/Астрал/Такском) ИЛИ облачная КЭП (КриптоПро DSS / СБИС / Контур —
  ключ в HSM, подпись по OTP, токен не нужен, законна по 63-ФЗ). Облачная удобна при многих клиентах.
- Спецоператор в режиме «обслуживающая бухгалтерия» (много клиентов, права по сотрудникам,
  пакетная отправка, единый календарь, входящие требования ФНС):

| Оператор | Ориентир 2026 |
|---|---|
| Контур.Экстерн (ОБ) | пакет «Учётный» ~45 600 ₽/год; есть модуль удалённого подписания |
| СБИС/Saby (уполн. бухгалтерия) | поотчётно 15–90 ₽/отчёт + подключение ~3 500 ₽ |
| 1С-Отчётность (Калуга-Астрал) | из 1С; представитель 15–70 ₽/отчёт |
| Такском | индивидуально, от ~5 000 ₽/год за организацию |

Все поддерживают МЧД и подпись представителя.

## ЧАСТЬ 4. Получение первички и данных удалённо

- Банковская выписка: (1) DirectBank — прямой канал банк↔1С, 40+ банков; (2) роль «Бухгалтер/просмотр»
  в интернет-банке клиента (Сбер Бизнес, Т-Бизнес, Альфа — онлайн); (3) ручной файл 1CClientBankExchange
  (СвояКнига уже импортирует, Win-1251).
- ЭДО контрагентов: роуминг (Диадок/СБИС) — видите счета-фактуры/УПД клиента у себя.
- Онлайн-касса: клиент добавляет вас в кабинет ОФД → все чеки/выручка, выгрузка в учётную систему.
- Сканы/фото: ЛК сервиса / чат / облако (юр. силу даёт только ЭДО с КЭП).

## ЧАСТЬ 5. Оплата налогов за клиента (ЕНС)

1. Безопасно: вы готовите реестр → клиент платит сам в своём банке.
2. Доступ к банку клиента: вы создаёте платёжки → клиент подтверждает (push/2FA).
3. Оплата за третье лицо со счёта конторы (ст. 45 НК — можно).

ВАЖНО: с 01.09.2026 (ФЗ-425 от 28.11.2025) пополнять чужой ЕНС «впрок» нельзя — зачёт только при
текущем долге, иначе возврат. → С осени 2026 основной режим: клиент платит сам по вашему реестру.
Источник: https://www.klerk.ru/buh/news/673684/

## ЧАСТЬ 6. Оформление отношений дистанционно

- Договор: ЭДО (полная юр. сила) / сканы + оригиналы почтой / оферта + акцепт.
- Прописать: перечень услуг и дедлайны первички; лимит ответственности конторы (+ страховка);
  доступы (банк/ОФД/ЛК ФНС) и их отзыв; обязанность выдать МЧД за N дней и перевыпустить при смене
  директора; поручение на обработку ПДн по 152-ФЗ (оператор ПДн — клиент).

## ЧАСТЬ 7. Онбординг далёкого клиента — чек-лист (1–2 недели)

- Дни 1–2: анкета, КП, выбор модели (Б).
- Дни 2–4: договор по ЭДО/сканам, аванс.
- Дни 3–5: клиент получает КЭП ФНС в своём банке → выдаёт МЧД (ФНС/Росстат + СФР).
- Дни 4–6: доступы — роль «бухгалтер» в банке, DirectBank, пользователь ОФД, роуминг ЭДО.
- Дни 5–7: аудит — база/сальдо ЕНС, список «хвостов» (несданное, долги).
- Дни 7–10: регламент первички, занос остатков, банковский обмен.
- Дни 10–14: первый отчёт → подпись КЭП + МЧД → квитанция ФНС → уведомили клиента.

CRM для бухфирм: Мультибухгалтер (от 700 ₽/мес).

## ЧАСТЬ 8. Подводные камни 2026

- 01.04.2026: ФНС на ГОСТ 34.12-2018 → КриптоПро CSP 5.0+, Рутокен ЭЦП 3.0 (2.0 не принимается).
- 01.02.2026: в МЧД обязательны срок и текстовое наименование полномочия (приказ Минцифры №1001 от 05.11.2025).
- 01.09.2026: СФР — только сертификаты с параметром «Согласование ключей» (до 28.08.2025 заменить) + ЕНС-ограничение.
- Отзыв МЧД при уходе бухгалтера/смене директора — немедленно.

## Рекомендация под кейс Воронеж → Владивосток

Модель Б. Клиент: КЭП ФНС в местном Сбере/ВТБ или «Моя подпись»; две МЧД (ФНС/Росстат + СФР) через
Контур.Экстерн/СБИС; роль «просмотр» в интернет-банке + пользователь ОФД. Бухгалтер: облачная/токенная
КЭП физлица + оператор «обслуживающая бухгалтерия». Налоги — клиент по реестру. Договор — по ЭДО.

## Что встроить в СвояКнигу (приоритет)
1. Реальная отправка через API спецоператора (Контур/СБИС/Астрал) ИЛИ режим «подготовщик XML» +
   отправка в кабинете оператора (XML по формам уже генерится).
2. Подпись: интеграция КриптоПро (браузерный плагин) для токенной КЭП или облачная подпись по API
   вместо текущей имитации.
3. Реестр МЧД в карточке клиента: номер/ведомство/срок/статус + алерт «МЧД истекает».
4. Мультиклиентность (кабинет обслуживающей бухгалтерии).
