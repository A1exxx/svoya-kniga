# Бесплатный тест «боевого режима» — пошагово

Цель: бухгалтер заходит по логину/паролю с любого устройства, данные сохраняются
на сервере с историей. **Бесплатно, без карты, на ВЫДУМАННЫХ данных** (реальные
ПДн — только РФ-хостинг, см. SERVER-SETUP.md).

Нужно создать 2 бесплатных аккаунта (это делает владелец — Claude не может
создавать аккаунты). ~15 минут.

## Шаг 1. База данных — Neon (бесплатно, без карты)
1. Зайти на neon.tech → Sign up (через GitHub).
2. Create project → регион Europe.
3. Скопировать строку подключения (Connection string), вид
   `postgresql://user:pass@ep-xxx.eu-central-1.aws.neon.tech/neondb?sslmode=require`.
4. **Заменить начало** `postgresql://` → `postgresql+psycopg://` (драйвер SQLAlchemy).
   Получится `postgresql+psycopg://user:pass@.../neondb?sslmode=require` — это `DATABASE_URL`.

## Шаг 2. Бэкенд — Render (бесплатно, без карты для web-сервиса)
1. Зайти на render.com → Sign up (через GitHub).
2. New → **Blueprint** → выбрать репозиторий `A1exxx/svoyakniga`.
   Render прочитает `render.yaml` и создаст сервис `svoyakniga-api`.
3. В переменной **DATABASE_URL** вставить строку из Шага 1.
4. Apply / Deploy. Через ~3–5 минут будет адрес вида
   `https://svoyakniga-api.onrender.com`.
5. Проверка: открыть `https://svoyakniga-api.onrender.com/api/health` → `{"status":"ok"}`.

> Бесплатный Render «засыпает» после простоя — первый вход после паузы грузится
> ~30–60 сек, дальше быстро. Для теста это нормально.

## Шаг 3. Подключить фронтенд (это делает Claude)
Прислать Claude адрес бэкенда (`https://svoyakniga-api.onrender.com`). Claude:
1. Соберёт фронтенд с `VITE_API_BASE=<адрес>`.
2. Задеплоит на GitHub Pages (или отдельный адрес).
3. После этого https://a1exxx.github.io/svoyakniga/ покажет **экран входа**, и
   бухгалтер сможет завести аккаунт.

## Что уже проверено локально
Регистрация → вход → данные с любого устройства → автосохранение → история версий
→ откат → экспорт. 7 тестов бэкенда + ручная проверка в браузере.

## Когда перейти на реальные данные
Как только нужны настоящие клиенты: переехать на **РФ-хостинг** (Timeweb/Яндекс),
`pg_dump`→`pg_restore` базы, сменить `DATABASE_URL`, подать уведомление в РКН.
Код и схема те же — переезд за час (см. SERVER-SETUP.md).
