# Локальный запуск на своём компьютере (боевой режим, база на вашем диске)

Это полноценная версия с **входом по логину/паролю** и **базой данных**, но всё
работает на вашем компьютере — без интернета, без оплаты, без юридических
формальностей. Данные лежат в файле `backend\svoyakniga.db` на вашем диске.

Когда функционал утвердим — ту же базу и код перенесём на платный защищённый
РФ-сервер (см. `SERVER-SETUP.md`); переезд — `pg_dump`/импорт, код тот же.

## Как запустить
1. Двойной клик по **`start-svoyakniga.bat`** в папке `D:\svoya-kniga`.
2. Откроется браузер на `http://localhost:8011`.
3. Первый раз — «Создать кабинет» (любой email как логин + пароль). Дальше — «Войти».
4. Работаете как обычно — всё **сохраняется автоматически** на сервере (в локальной
   базе), с историей версий и откатом (панель «☁» слева внизу).

Сервер крутится в **свёрнутом окне** (python). Чтобы остановить — закройте это окно.

## Где данные
- База: `D:\svoya-kniga\backend\svoyakniga.db` (один файл — можно копировать как бэкап).
- В кабинете: кнопка **«Скачать»** (☁-панель) выгружает весь кабинет в JSON.

## Доступ с телефона/другого устройства в той же сети (необязательно)
По умолчанию сервер виден только на этом компьютере (127.0.0.1). Чтобы открыть
с телефона в той же Wi-Fi: запустить сервер с `--host 0.0.0.0` и заходить по
`http://<IP-компьютера>:8011`. Для боевого использования с реальными данными
это не нужно — лучше сразу РФ-сервер.

## Если приложение обновили (новый код)
Удалить папку `web\dist-local` и снова запустить `.bat` — он пересоберёт фронтенд.

## Требования
Node.js и Python уже установлены (проект на них собран). Если `.venv` бэкенда
отсутствует — создать и поставить зависимости:
```
cd D:\svoya-kniga\backend
python -m venv .venv
.venv\Scripts\python -m pip install -r requirements.txt
```

⚠️ Локальная версия — для работы/теста на этом компьютере. Для нескольких
устройств и доступа извне нужен сервер (бесплатный тест: `DEPLOY-FREE-TEST.md`;
боевой РФ: `SERVER-SETUP.md`).
