# PAYROLL-NOTES — особенности зарплатного модуля

## 1. Аванс / расчёт — декомпозиция НДФЛ

Начиная с 2023 НДФЛ перечисляется дважды в месяц. Для уведомлений ЕНС (КНД 1110355) каждая зарплата разбивается на:

- **advance_ndfl** — НДФЛ с аванса (выплата 1–22-го числа)
- **settlement_ndfl** — НДФЛ с расчёта (выплата 23-го — конец месяца)

**Инвариант:** `advance_ndfl + settlement_ndfl == ndfl` (итоговый НДФЛ за месяц).

Расчёт ведётся нарастающим итогом с начала года (ст. 226 НК), поэтому аванс считается по тем же прогрессивным ступеням, что и расчёт — просто с меньшей долей дохода.

**Вычеты на детей** применяются на этапе расчёта (не авансом), аналогично поведению 1С:

- При расчёте аванса вычеты не учитываются → сумма аванса чуть больше реальной половины месячного НДФЛ.
- При расчёте итогового НДФЛ вычеты применяются к полному доходу за месяц → settlement_ndfl может быть меньше advance_ndfl.

Логика: `payroll.ts` → `splitNdflByPeriods(gross, advanceFraction, yearToDate, params)`.
Уведомления: `notifications.ts` → `buildNotifications(year, month, employees, payrollResults)`.

## 2. Авто-стаж из даты приёма

Трудовой стаж влияет на коэффициент оплаты больничных (60% / 80% / 100%).

**Источник стажа** (приоритет):
1. `employee.stazh_years` — **ручной ввод** (если бухгалтер вводит стаж явно)
2. Автоматически из `employee.hire_date` — число полных лет от даты приёма до даты начала больничного

Модуль: `web/src/lib/stazh.ts`

```
calcStazh(hireDate: string, asOf: string): number   // полных лет
sickCoeff(stazhYears: number): 0.6 | 0.8 | 1.0
```

Экран «Карточка сотрудника» показывает «Стаж: X лет (авто)» или «X лет (ручной)» в зависимости от того, что заполнено.

При добавлении нового сотрудника через форму `hire_date` обязателен → стаж рассчитывается автоматически и отображается до ввода явного значения.

## 3. Заработок по годам — базы для отпускных и больничных

Для корректного расчёта:
- **Отпускных** — нужен заработок за последние 12 месяцев
- **Больничных** — нужен заработок за 2 предшествующих календарных года

Эти суммы хранятся в `employee.earningsByYear: Record<number, number>` (стор `employeesStore`).

Заполняется:
- При первичном вводе (бухгалтер вводит вручную для «исторических» годов)
- Автоматически из операций в `opsStore` начиная с даты регистрации ИП / начала ведения учёта в СвояКниге

Модуль: `web/src/lib/earnings.ts`

```
getEarningsBase(employee, years: number[]): Decimal   // сумма за указанные годы
calcVacationDailyAverage(employee, year): Decimal     // ÷ 12 ÷ 29,3
calcSickDailyAverage(employee, year): Decimal         // ÷ 730, с лимитами
```

**Ограничение для больничных:**
- Минимум: `МРОТ × 24 ÷ 730` (защита от нуля)
- Максимум: `(SICK_LEAVE_BASE_BY_YEAR[y-1] + SICK_LEAVE_BASE_BY_YEAR[y-2]) ÷ 730`

`SICK_LEAVE_BASE_BY_YEAR` → `payroll.ts` (актуальные базы по годам, нужно обновлять ежегодно).

## 4. Распределение НДФЛ по периодам уведомлений

Модуль `notifications.ts` формирует уведомления КНД 1110355 за год. Каждый месяц порождает до двух записей:

| Выплата | Период уведомления | КБК |
|---------|-------------------|-----|
| Аванс (1–22-го) | 2-я половина того же месяца | НДФЛ КБК |
| Расчёт (23-го – конец) | 1-я половина следующего месяца | НДФЛ КБК |

Декабрьская зарплата — особый случай: расчёт за 23–31 декабря включается в уведомление за декабрь (не январь следующего года), чтобы не переходить год.

**Инвариант годовой суммы:** сумма НДФЛ по всем строкам уведомлений за год == сумма НДФЛ по всем сотрудникам за год. Тест: `notifications.test.ts`.
