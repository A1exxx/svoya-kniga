## 1. Что это и для кого

Веб-приложение бухгалтерии для **ИП на УСН** (Россия) — узкий точный домен,
не «ещё один ERP», без двойной бухгалтерской записи. Делает то же, что
Контур.Эльба для предпринимателя: считает налоги/взносы, ведёт операции и
первичку, формирует отчётность, считает зарплату сотрудников, напоминает о сроках.

**Принципы:** сначала функционал, который считает ПРАВИЛЬНО (это калькулятор —
ошибаться нельзя), потом удобство; все ставки/лимиты — параметры по годам;
корректность сверяет бухгалтер. Интерфейс — близко к Эльбе (левое меню,
«Актуальные задачи»), чтобы привыкший бухгалтер ориентировался сразу.

| `/contractors` | Контрагенты | `Contractors.tsx` | справочник контрагентов, автозаполнение по ИНН (DaData/демо-словарь) |
| `/goods` | Товары | `Goods.tsx` | номенклатура товаров/услуг (пикер в документах) |

| `/requisites` | Реквизиты | `Requisites.tsx` | данные ИП, банк, КЭП, брендинг (логотип/подпись/печать), система налогообложения (объект УСН, НДС-ставка, АУСН, торговый сбор) |

**Сотрудники — подробно (вкладки `Employees.tsx`):**
- **Штат** — карточка сотрудника = единый источник (оклад, дети-вычеты, дата приёма, стаж авто/вручную, аванс %, заработок по годам помесячно, личные данные, увольнение), печать кадровых форм (личная карточка Т-2, приказ о приёме Т-1, справка НДФЛ, заявление на вычет).
- **Зарплата** — помесячный расчёт по рабочим дням (производственный календарь); **начисление ПО ФАКТУ** (месяц появляется после «Начислить», не заполняется вперёд); столбец **«Выплата»** (отметка выдачи по месяцам); переключатель **авто-учёта больничных/отпусков без оплаты**.
- **Отпуск** — события (период+вид: очередной/по уходу/без оплаты), накопленные дни, приказ Т-6, график Т-7.
- **Больничный / Пособия** — события по датам → пособие (СДЗ за 2 года, стаж-коэффициент, МРОТ-пол подённо, потолок для стажа <6 мес).
- **Алименты** — галочка на сотруднике → авто-удержание (доля 1/4·1/3·1/2 или твёрдая сумма, лимит 70%).
- **Сводка по штату** — ФОТ/НДФЛ/взносы по всем, расчётно-платёжная ведомость.
- **Отчёты** — 6-НДФЛ, РСВ, ЕФС-1, перссведения (заполняются из штата).

## 4. База данных и хранение

Данные хранятся в трёх местах в зависимости от режима. **Один и тот же формат**
(JSON по ключам) переносится между ними (экспорт/импорт, синхронизация).

### 4.1 Локально (демо и офлайн-кэш)
Браузерный `localStorage` + зеркало в `IndexedDB` (на случай очистки localStorage).
Ключи (`web/src/state/*` + `lib/storage/`):

| Ключ localStorage | Стор / модуль | Содержимое |
|---|---|---|
| `svoyakniga.orgs.v2` | `orgStore` | список ИП + активный ИП (реквизиты, налогообложение, финансы) |
| `svoyakniga.ops.v1` | `opsStore` | операции доход/расход (КУДиР) |
| `svoyakniga.docs.v1` | `docsStore` | документы (счета/акты/накладные/УПД/договоры) |
| `svoyakniga.payments.v1` | `paymentsStore` | платёжные поручения |
| `svoyakniga.contractors.v1` | `contractorsStore` | контрагенты |
| `svoyakniga.goods.v1` | `goodsStore` | товары/услуги |
| `svoyakniga.employees.v1` | `employeesStore` | сотрудники (всё: оклад, отпуска, больничные, начисления, выдача, алименты) |
| `svoyakniga.archive.v1` | `archiveStore` | архив сданного (со снимком данных) |
| `svoyakniga.taxoffice.v1` | `taxOfficeStore` | ЕНС-сальдо, сверки, письма ФНС/СФР |
| `svoyakniga.paramOverrides.v1` | `paramsStore` | ручные правки параметров по годам |
| `svoyakniga.snapshots.v1` | `storeAdmin` | снимки для отката |
| `svoyakniga.audit.v1` | `storeAdmin` | журнал действий (создание/изменение/удаление) |
| `svoyakniga.theme` / `svoyakniga.errorlog.v1` / `svoyakniga.dadata.token` / `svoyakniga.automation.v1` / `svoyakniga.lastAutoSnapshot` | служебные | тема, журнал ошибок, ключ DaData, настройки автоматизации, метка автоснимка |

Каждый стор — Context-провайдер (`web/src/state/*.tsx`), данные **по orgId**
(`Record<orgId, T[]>`), пишутся через `persistKey` (localStorage + IDB).
Изменения логируются через `logChange`/`diffFields` → виден в Админке.
Защита от потери: автоснимок раз в сутки, ручные снимки, экспорт/импорт `.json`.
Подробно — [STORAGE-AND-AUDIT.md](STORAGE-AND-AUDIT.md).

> `web/src/state/store.tsx` (`inputs.v1`) — мёртвый legacy, не смонтирован.

### 4.2 На сервере (боевой режим)
PostgreSQL (или SQLite локально). Схема — `backend/app/models.py`:

| Таблица | Поля (ключевые) | Назначение |
|---|---|---|
| `users` | id, email (уникальный логин), password_hash (Argon2id), name, created_at | аккаунты |
| `workspaces` | id, user_id, name, current_version | «рабочий стол» пользователя (1 на юзера = весь кабинет) |
| `workspace_versions` | id, workspace_id, version, data (JSON), size_bytes, note, saved_at | **история ревизий** — каждое сохранение = новая неизменяемая запись (откат к любой); уникальность (workspace_id, version) |
| `sessions` | id, token, user_id, expires_at | серверные сессии (httpOnly-cookie, отзыв при выходе) |
| `audit_log` | id, user_id, action, detail, at | журнал входов/регистраций |

В `workspace_versions.data` — тот же JSON, что локальный бэкап (все ИП,
операции, документы, сотрудники). Бэкенд хранит ВЕСЬ кабинет как документ с
версиями, а не разбивает по таблицам — это даёт быстрый старт + историю/откат;
эволюция к реляционной схеме возможна позже.

### 4.3 Многослойная защита «чтобы ничего не потерялось»
1. История версий в БД (на каждое сохранение). 2. PITR у managed-Postgres
(откат к секунде). 3. Автоснапшоты провайдера. 4. Ежедневный `pg_dump`
(`backend/scripts/backup.sh`). 5. Кнопка «Скачать» в интерфейсе. Локально — снимки
+ IDB-зеркало + экспорт.

## 5. Как считаются налоги (движок taxcore)

Расчёты живут в **двух зеркальных движках** с проверкой паритета тестами:
- TS: `web/src/lib/taxcore/*.ts` (работает в браузере, decimal.js)
- Python: `backend/taxcore/*.py` (тот же расчёт; FastAPI отдаёт через `/api/calc`)

| Модуль (ts/py) | Что считает |
|---|---|
| `usn` | УСН: доходы 6% / Д−Р 15%, авансы поквартально нарастающим итогом, минимальный налог 1%, уменьшение на взносы |
| `vznosy` | страховые взносы ИП: фиксированные + 1% свыше 300 000 ₽, пропорция по дате регистрации (неполный год) |
| `payroll` | зарплата: НДФЛ прогрессией 13/15/18/20/22% нарастающим, аванс-декомпозиция, детские вычеты, страховые взносы с ФОТ (+МСП), травматизм; отпускные (СДЗ/29,3); больничные (СДЗ за 2 года, стаж, МРОТ-пол/потолок); алименты (доля−НДФЛ, 70%) |
| `vat` | НДС на УСН: освобождение / 5% / 7% / 10% / общая (20%→22% с 2026); исчисленный (output) и к уплате (минус вычет) |
| `patent` | ПСН: ПВГД×6%×срок − взносы (потолок 50% с работниками) |
| `ens` | налоговый календарь: даты авансов/деклараций/уведомлений/взносов/НДС (3 доли)/зарплатных обязанностей; сдвиг на рабочий день (производственный календарь) |
| `notifications` | уведомления КНД 1110355: коды периода (21/01…34/13), даты, привязка к КБК |
| `kbk` | справочник КБК (НДФЛ по ступеням, взносы, УСН, травматизм) |
| `params` | **параметры по годам**: ставки, лимиты, МРОТ, предельные базы, фикс.взносы, пороги НДС |
| `money` | производственный календарь (WORKDAYS_BY_YEAR, HOLIDAYS_BY_YEAR), рабочие дни месяца, сдвиг сроков |

Над движком: `web/src/lib/compute.ts` собирает расчёт по ИП из операций
(поквартально), `payrollSummary.ts` агрегирует штат, `vatBooks.ts` строит книги
продаж/покупок. Каждое число — с основанием в [TAX-RULES.md](TAX-RULES.md);
зарплатные тонкости — [PAYROLL-NOTES.md](PAYROLL-NOTES.md).

## 6. Серверный слой: аккаунты, API, синхронизация

Включается флагом `VITE_SERVER_MODE=true` (или `VITE_API_BASE=<url>`). Бэкенд —
`backend/app/`.

**Эндпоинты** (FastAPI, см. `http://localhost:8000/docs`):
- `POST /api/auth/register` · `POST /api/auth/login` · `POST /api/auth/logout` · `GET /api/auth/me` — аккаунты, сессия в httpOnly-cookie.
- `GET /api/workspace` — текущий кабинет · `PUT /api/workspace` — сохранить (новая версия) · `GET /api/workspace/versions` — история · `GET /api/workspace/versions/{v}` · `POST /api/workspace/restore/{v}` — откат · `GET /api/workspace/export` — скачать.
- `GET /api/health`, `/api/params`, `POST /api/calc`, `POST /api/calc/periods` — расчёты taxcore.

**Безопасность:** пароли Argon2id (OWASP-профиль), серверные сессии (отзыв при
выходе), лимит 5 попыток/15 мин, изоляция данных между пользователями, cookie
httpOnly (+Secure/SameSite=none в проде через env).

**Синхронизация (фронтенд `state/authStore.tsx` + `lib/cloudSync.ts`/`serverApi.ts`):**
при входе — подтяжка с сервера (сервер = источник правды; первый вход заливает
локальное наверх), автосохранение раз в 8 c при изменениях и при уходе со вкладки,
сбор/применение кабинета через те же `exportBackup`/`importBackup`, что и локальный
бэкап. Конкуренция версий защищена уникальным ограничением + повтором.

Архитектура и 152-ФЗ — [SERVER-SETUP.md](SERVER-SETUP.md); реальная ФНС —
[FNS-INTEGRATION.md](FNS-INTEGRATION.md).

`models.py`, `security.py`, `db.py`. **`backend/taxcore/`** — Python-движок (зеркало).

## 8. Печатные формы и XML-отчётность

Печать — через `PrintModal`; компоненты в `web/src/components/`:

| Форма | Компонент | Примечание |

## 9. Параметры по годам (как менять числа)

Все ставки/лимиты/взносы меняются ежегодно — вынесены в параметры:
- Код: `web/src/lib/taxcore/params.ts` (+ `backend/taxcore/params.py`, паритет).
- Через интерфейс: экран **Настройки** → «Параметры налогов по годам» (правки
  кладутся в `paramOverrides.v1`, применяются поверх кода без пересборки).

Что параметризовано: ставки УСН, лимиты доходов, МРОТ, предельные базы взносов,
фиксированные взносы ИП и потолок 1%, пороги и ставки НДС, вычеты на детей.
Полный список с основаниями — [CONFIGURABLE-PARAMS.md](CONFIGURABLE-PARAMS.md) и
[TAX-RULES.md](TAX-RULES.md).

## 10. Тесты и проверка корректности

- Фронт: `cd web && npx vitest run` — **351 тест** (расчёты, паритет-инварианты, формы, property-тесты fast-check).
- Бэк: `cd backend && .venv\Scripts\python -m pytest -q` — **97 тестов** (движок + аккаунты/синхронизация).
- E2E: Playwright (`web/e2e/`, `npm run test:e2e` после установки).
- Инвариант паритета: TS и Python дают одинаковые числа; ключевые инварианты
  (аванс+расчёт НДФЛ == месячный; база+налог == выручка) проверяются на случайных данных.
- Корректность периодически проверяется **многоагентными аудитами** против НК РФ
  (методология — [METHODOLOGY.md](METHODOLOGY.md)).

## 11. Что «по-настоящему», а что имитация

| По-настоящему работает | Имитация / демо (честно помечено) |
|---|---|
| Все расчёты налогов/взносов/зарплаты | Отправка деклараций/уведомлений в ФНС (квитанция-имитация) |
| Формирование печатных форм и XML | Сальдо ЕНС, сверка, требования ФНС (ручной ввод, ст.102 НК — налоговая тайна) |
| Учёт операций, первичка, документы | Подпись КЭП (хранится только описание сертификата) |
| Импорт банк-выписки (файл 1С) | Авто-подтяжка из банка по API (серверный этап) |
| Аккаунты, версии, бэкапы (серверный режим) | Проверка контрагента онлайн без ключа DaData |

Причина имитаций — регуляторика: реальная сдача в ФНС идёт по ТКС через
аккредитованного оператора ЭДО с КЭП. Это серверный этап (см. FNS-INTEGRATION.md).

## 12. Как сделать X (частые задачи)

- **Изменить ставку/лимит на год** → Настройки → параметры по годам (или `params.ts`).
- **Завести второй ИП** → список ИП слева → «+ Добавить ИП» → Реквизиты.
- **Удалить ИП / операцию / документ / сотрудника** → ✕/«удалить» с подтверждением (откат — снимок в Админке).
- **Посмотреть сколько НДС/УСН за месяц** → Деньги → карточка «Налоги по месяцам».
- **Начислить зарплату за месяц** → Сотрудники → Зарплата → выбрать месяц → «Начислить»; «Выплатить» — отметка выдачи.
- **Откатить данные** → Админ (локально) или панель «☁» → История (серверный режим).
- **Сделать бэкап** → Админ → «Скачать резервную копию» (локально) или «☁» → Скачать.
- **Запустить боевой режим на ПК** → `start-svoyakniga.bat`.
- **Перенести базу на сервер** → `pg_dump`/импорт + смена `DATABASE_URL` (SERVER-SETUP.md).
- **Добавить новую печатную форму** → новый компонент в `components/`, открыть через `PrintModal`.
- **Добавить параметр расчёта** → `taxcore/params.ts` + `params.py` (паритет) + тест.
