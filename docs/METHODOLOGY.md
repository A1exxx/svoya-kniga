# Методология корректности расчётов

«СвояКнига» — это калькулятор: формула пишется один раз и дальше считает детерминированно,
поэтому **ошибаться нельзя**. Этот документ описывает, как мы обеспечиваем и проверяем
корректность. Конкретные числа и их правовые основания — в [TAX-RULES.md](./TAX-RULES.md).

## Принципы

1. **Каждое число — с основанием.** У каждого параметра задокументировано «почему именно так»:
   статья НК РФ / номер ФЗ + источник + человеческое объяснение. Видно в коде (`params.ts`/`payroll.py`
   поле `note`) и на экране «Настройки → параметры налогов по годам».
2. **Актуальность.** Законы меняются ежегодно. Параметры версионированы по годам
   (`YEARS`, `PAYROLL`, `getKbk`). 2026 учитывает реформу НДС (ФЗ № 425-ФЗ): общая ставка 22%,
   порог освобождения 20 млн.
3. **Проверяем многократно, разными способами** (см. ниже).

## Способы проверки

### 1. Два независимых движка с паритетом
Все формулы реализованы дважды: TypeScript (`web/src/lib/taxcore/*.ts`, decimal.js, для UI) и
Python (`backend/taxcore/*.py`, Decimal, эталон). Тесты сверяют, что результаты совпадают
1-в-1. Расхождение = регрессия. Запуск: `cd web && npx vitest run`; `cd backend && python -m pytest -q`.

### 2. Юнит-тесты на известных сценариях
Эталонные значения посчитаны вручную и зафиксированы (НДФЛ-ступени, взносы, УСН-авансы,
мин.налог, НДС 5/7/22%, отпускные/больничные/алименты, коды периодов уведомлений).

### 3. Property-тесты (инварианты на случайных данных)
`web/src/lib/taxcore/invariants.property.test.ts` (fast-check, 200+ прогонов на свойство):
- **НДФЛ-декомпозиция:** `advance_ndfl + settlement_ndfl == ndfl` для любого оклада и % аванса;
  обе части ≥ 0.
- **Уведомления:** сумма НДФЛ-строк за год == `ndfl_year` (декомпозиция, не новый итог).
- **НДС:** `база + налог == выручка` (с НДС, ±1 ₽ на округление), налог ≥ 0.

### 4. Деньги — в Decimal, округление по правилам
`round_rub` (целые рубли, ROUND_HALF_UP) для налога; `money` (копейки) для взносов и сумм.
Агрегация доходов/расходов — в Decimal (без float-дрейфа).

### 5. Сверка с бухгалтером
Финальную корректность перед реальной сдачей подтверждает бухгалтер пользователя на данных
реальных ИП. Значения со статусом `verified` помечены; **дефлятор-пороги НДС-2026 (5/7%) требуют
подтверждения** при публикации боевой версии.

## Что НЕ гарантируется автоматически (сверять вручную)
- Точные XSD-форматы ФНС для XML (генерация демонстрационная — сверять в «Налогоплательщик ЮЛ»).
- Дефлятор НДС-2026 для границ спец-ставок (5/7%).
- Применимость льготы МСП (зависит от ОКВЭД/реестра).
- Реальная отправка в ФНС (имитация; реальная — через оператора ЭДО, см. [FNS-INTEGRATION.md](./FNS-INTEGRATION.md)).
