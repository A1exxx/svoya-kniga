# HOW-IT-WORKS — архитектура СвояКниги

## Стек

- **Фронтенд:** React 19 + Vite + TypeScript + Tailwind CSS v3
- **Маршрутизация:** HashRouter (совместимо со статическим хостингом GitHub Pages)
- **Base URL:** `'./'` — деплой на любой путь без nginx-rewrite
- **Тестирование (TS):** Vitest
- **Тестирование (Python):** pytest

## Паритет расчётных движков TS ↔ Python

Все налоговые расчёты реализованы **дважды**, независимо:

| Слой | Путь | Используется |
|------|------|-------------|
| TypeScript (UI) | `web/src/lib/taxcore/*.ts` | React-приложение, продакшен |
| Python | `backend/taxcore/*.py` | Верификация чисел, pytest |

Оба слоя должны давать побитово идентичные результаты на одинаковых входных данных. При расхождении — Python-реализация является арбитром (ссылается на НК напрямую). При добавлении нового налогового правила — изменения вносятся в оба файла одновременно.

**Округление:**
- `round_rub` — целые рубли (для налоговых сумм, УСН)
- `money` — копейки (для взносов ИП, зарплатных выплат)

Обе функции используют `decimal.js` / `Decimal` (Python) для исключения float-дрейфа.

## Карта модулей taxcore

| Модуль | TS-файл | Что считает |
|--------|---------|-------------|
| `params` | `taxcore/params.ts` | `YearParams` — все числовые константы по годам (взносы, лимиты, ставки УСН, НДС). Источник правды для остальных модулей. |
| `usn` | `taxcore/usn.ts` | Авансы УСН по кварталам, годовой налог, минимальный налог (1%), вычет взносов |
| `vznosy` | `taxcore/vznosy.ts` | Фиксированные взносы ИП «за себя» (фикс + 1%), пропорция за неполный год |
| `payroll` | `taxcore/payroll.ts` | НДФЛ (прогрессивная шкала), страховые взносы с ФОТ, МСП-льгота, аванс/расчёт-разбивка, отпускные, больничные, алименты |
| `vat` | `taxcore/vat.ts` | НДС для УСН: освобождение / спец-ставки 5% / 7% / общая ставка |
| `patent` | `taxcore/patent.ts` | Расчёт стоимости патента (ПСН) |
| `ens` | `taxcore/ens.ts` | Календарь ЕНС, даты уведомлений и платежей |
| `notifications` | `taxcore/notifications.ts` | Формирование уведомлений КНД 1110355, распределение НДФЛ по периодам |
| `kbk` | `taxcore/kbk.ts` | Справочник КБК (УСН, взносы, НДФЛ, НДС, ЕНС) |
| `money` | `taxcore/money.ts` | Примитивы `money` / `round_rub` / `toDecimal` |

Вспомогательные модули (в `web/src/lib/`):

| Модуль | Что делает |
|--------|-----------|
| `compute.ts` | Агрегирует операции из `opsStore` → базы для УСН/НДС/КУДиР |
| `stazh.ts` | Авто-расчёт трудового стажа из даты приёма → коэффициент больничных (0.6 / 0.8 / 1.0) |
| `earnings.ts` | Заработок сотрудника по годам → база для среднедневного (отпускные ÷ 29,3; больничные ÷ 730) |
| `payrollSummary.ts` | Сводка по штату: НДФЛ, взносы, чистые выплаты всех сотрудников за период |
| `declarationXml.ts` | XML-выгрузка декларации УСН (КНД 1152017) |
| `ensXml.ts` | XML-выгрузка уведомления ЕНС (КНД 1110355) |
| `vatDeclarationXml.ts` | XML-выгрузка декларации НДС |
| `bankImport.ts` | Парсер выписок 1CClientBankExchange (Win-1251) |
| `innLookup.ts` | Автозаполнение реквизитов контрагента по ИНН (DaData + демо-словарь) |
| `numberToWords.ts` | Сумма прописью (рубли) — для первичных документов |
| `format.ts` | Форматирование денег, дат, номеров |
| `download.ts` | Триггер скачивания XML/PDF из браузера |

## Сторы (React Context + localStorage)

| Стор | Ключ localStorage | Что хранит |
|------|-------------------|-----------|
| `orgStore` | `svoyakniga.org.<orgId>` | Реквизиты ИП: ИНН, ОГРНИП, адрес, ОКТМО, ставка НДС |
| `opsStore` | `svoyakniga.ops.<orgId>` | Операции (доходы/расходы) — основа КУДиР и расчёта УСН |
| `docsStore` | `svoyakniga.docs.<orgId>` | Исходящие/входящие документы (счета, акты, накладные, УПД) |
| `contractorsStore` | `svoyakniga.contractors.<orgId>` | Контрагенты с реквизитами |
| `goodsStore` | `svoyakniga.goods.<orgId>` | Номенклатура товаров/услуг |
| `employeesStore` | `svoyakniga.employees.<orgId>` | Сотрудники: реквизиты, стаж, заработок по годам, дети |
| `archiveStore` | `svoyakniga.archive.<orgId>` | Архив закрытых задач/событий |
| `taxOfficeStore` | `svoyakniga.taxoffice.<orgId>` | Данные раздела «Налоговая»: сальдо ЕНС, запросы сверки, письма |
| `paramsStore` | `svoyakniga.paramOverrides.v1` | Пользовательские переопределения `YearParams` (глобальный ключ, не скоупирован по orgId) |

Все данные скоупированы по `orgId` — можно вести несколько ИП в одном браузере.

## UI-конвенции

Компоненты без внешних библиотек UI, только Tailwind:

- **Card** — белая карточка с тенью, контейнер раздела
- **Field** — строка «Метка: значение» в карточке
- **Note** — серый блок-подсказка (НК-основание, сноска)
- **Row** — строка таблицы с чередующимся фоном
- **PrintModal** — модальное окно с кнопкой «Печать» (window.print), управляет видимостью @media print

Деньги форматируются через `format.ts`: `formatRub(n)` → `«1 234 567,89 ₽»`.

## Команды разработки

```bash
# Фронтенд
cd web
npm install
npm run dev          # dev-сервер Vite на localhost:5173
npm run build        # сборка в web/dist/
npx vitest run       # все TS-тесты

# Бэкенд (расчётный движок Python)
cd backend
python -m pytest -q  # все Python-тесты

# Деплой на GitHub Pages
cd web
npm run build
npx gh-pages -d dist -b gh-pages --dotfiles
```

> После деплоя продакшен-версия доступна на `https://a1exxx.github.io/svoya-kniga/`
