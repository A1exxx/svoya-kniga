# CONFIGURABLE-PARAMS — где менять числа

## Главный источник: taxcore/params.ts + params.py

### `web/src/lib/taxcore/params.ts`

Структура `YearParams` — все налоговые константы одного года:

```ts
interface YearParams {
  year: number
  fixed_contributions      // фикс. взносы ИП «за себя» (Decimal, рубли)
  income_threshold_1pct    // порог 1% (300 000 ₽ — ст. 430 НК)
  rate_1pct                // ставка 1% (0.01)
  max_variable_contributions  // потолок доп. 1% (321 818 ₽ в 2026)
  usn_income_rate          // УСН «Доходы» базовая ставка (0.06)
  usn_income_minus_rate    // УСН «Д−Р» базовая ставка (0.15)
  usn_min_tax_rate         // минимальный налог (0.01)
  vat_exempt_threshold     // порог освобождения от НДС (20 000 000 в 2026)
  vat_rate5_limit          // верхняя граница 5% (272 500 000 в 2026)
  vat_rate7_limit          // верхняя граница 7% (490 500 000 в 2026)
  vat_general_rate         // общая ставка НДС (22 в 2026)
  verified                 // true — сверено с НК/бухгалтером
  note                     // текстовое основание (ст. НК, ФЗ)
}
```

Словарь `YEARS: Record<number, YearParams>` содержит 2024, 2025, 2026. При наступлении нового года — добавить новую запись. `verified: false` до подтверждения бухгалтером.

### `backend/taxcore/params.py`

Держать **синхронно** с `params.ts`. Те же поля, `Decimal` вместо `decimal.js`. При расхождении Python-реализация является арбитром (на ней построены pytest-тесты).

### `NDFL_TIERS: Array<[Decimal | null, Decimal]>` (константа, не в `PAYROLL`)

Шкала НДФЛ — 5 ступеней, `null` в первой позиции = верхний (бесконечный) порог. Менять только при изменении ст. 224 НК.

### `SICK_LEAVE_BASE_BY_YEAR: Record<number, Decimal>`

Предельные базы для расчёта больничных (сумма заработка за год). Обновлять ежегодно по Постановлению Правительства. Файл: `payroll.ts`, `payroll.py`.

## КБК: kbk.ts

`web/src/lib/taxcore/kbk.ts` — справочник кодов бюджетной классификации для:

- УСН (аванс, годовой, минимальный)
- Взносы ИП «за себя» (фикс + 1%)
- Взносы с ФОТ (ОПС, ОМС, ВНиМ, травматизм)
- НДФЛ (два КБК: «15%» и «выше 15%» для ступеней шкалы)
- НДС
- ЕНП (единый налоговый платёж)

КБК изменяются редко (проверять ежегодно на nalog.gov.ru). При изменении → обновить `kbk.ts` и соответствующий Python-словарь.

## Пользовательские переопределения: paramsStore

Экран «Настройки → параметры налогов» (`/settings`) → `paramsStore` (`web/src/state/paramsStore.ts`) хранит пользовательские правки `YearParams` поверх seed-значений из `params.ts`.

Это позволяет, например, задать региональную ставку УСН 1% вместо 6%, не трогая код.

Приоритет: `paramsStore overrides > YEARS[year]` — при каждом расчёте сначала берётся `getParams(year)`, затем мержатся данные из стора.
